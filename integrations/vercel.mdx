Turso works with Vercel serverless and edge functions through the [Turso Serverless JavaScript Driver](https://turso.tech/blog/introducing-turso-serverless-javascript-driver).

## Serverless JavaScript Driver

The `@tursodatabase/serverless` package enables connections to Turso databases from serverless and edge environments using only the `fetch` API:

```bash
npm install @tursodatabase/serverless
```

Configure your Vercel environment with the following variables:

- `TURSO_DATABASE_URL` — Your Turso database URL
- `TURSO_AUTH_TOKEN` — Your Turso auth token

### Usage

```typescript
import { connect } from "@tursodatabase/serverless";

const conn = connect({
  url: process.env.TURSO_DATABASE_URL,
  authToken: process.env.TURSO_AUTH_TOKEN,
});

const stmt = conn.prepare("SELECT * FROM users WHERE id = ?");
const row = await stmt.get([123]);
```

### LibSQL Compatibility

For compatibility with the libSQL client API:

```typescript
import { createClient } from "@tursodatabase/serverless/compat";

const client = createClient({
  url: process.env.TURSO_DATABASE_URL,
  authToken: process.env.TURSO_AUTH_TOKEN,
});

const result = await client.execute("SELECT * FROM users WHERE id = ?", [123]);
```

## @tursodatabase/vercel-experimental

<Warning>
This package is in beta. It may still contain bugs and unexpected behavior. Use caution with production data and ensure you have backups.
</Warning>

Zero-config SQLite databases for Vercel Functions with automatic sync.

### Features

- **Zero-config databases** — Databases are created automatically on first use
- **Local SQLite** — Fast reads from a local database copy in the serverless function
- **Automatic sync** — Writes are automatically pushed to Turso via Vercel's `waitUntil`
- **Partial sync** — Sync only the data you need for efficient cold starts

### Install

```bash
npm install @tursodatabase/vercel-experimental
```

### Setup

1. Get your Turso API token:

```bash
turso auth api-tokens mint my-vercel-token
```

2. Get your organization slug:

```bash
turso org list
```

3. Add environment variables to your Vercel project:

```
TURSO_API_TOKEN=your-api-token
TURSO_ORG=your-org-slug
TURSO_DATABASE=your-database-name
```

<Note>
Store `TURSO_DATABASE` as a secret environment variable in Vercel. The API token grants access to your entire Turso organization, so keeping the database name secret prevents injection attacks.
</Note>

### Quickstart

```ts
import { createDb } from "@tursodatabase/vercel-experimental";

const db = await createDb(process.env.TURSO_DATABASE!);

// Create tables
await db.execute(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT UNIQUE
  )
`);

// Insert data
await db.execute(
  "INSERT INTO users (name, email) VALUES (?, ?)",
  ["Alice", "alice@example.com"]
);

// Changes sync automatically via waitUntil - no manual push needed!

// Query data
const result = await db.query("SELECT * FROM users");
console.log(result.rows);
```

### API Reference

#### `createDb(name, options?)`

Creates or retrieves a database instance.

```ts
const db = await createDb(process.env.TURSO_DATABASE!, {
  group: "default",        // Database group (optional)
  partialSync: true,       // Enable partial sync (optional)
});
```

#### `db.query(sql, params?)`

Execute a SELECT query and return results.

```ts
const result = await db.query("SELECT * FROM users WHERE id = ?", [1]);

console.log(result.columns); // ["id", "name", "email"]
console.log(result.rows);    // [[1, "Alice", "alice@example.com"]]
```

#### `db.execute(sql, params?)`

Execute an INSERT, UPDATE, DELETE, or DDL statement.

```ts
await db.execute("UPDATE users SET name = ? WHERE id = ?", ["Bob", 1]);
```

#### `db.push()`

Manually push local changes to the remote Turso database. Optional since changes are automatically synced via Vercel's `waitUntil`.

```ts
await db.execute("INSERT INTO users (name) VALUES (?)", ["Charlie"]);
await db.push(); // Explicit sync
```

#### `db.pull()`

Pull latest changes from the remote Turso database.

```ts
await db.pull();
```
