## Prerequisites

Before you start, make sure you:

- [Install the Turso CLI]
- [Sign up or login to Turso]
- Have a Quasar app &mdash; [learn more]

<Steps>

<Step title="Install the libSQL SDK">

<CodeGroup>

```bash npm
npm install @libsql/client
```

```bash pnpm
pnpm add @libsql/client
```

```bash yarn
yarn add @libsql/client
```

</CodeGroup>

</Step>

<Step title="Configure database credentials.">

Get the database URL.

```bash
turso db show --url <database-name>
```
Get the database authentication token.

```bash
turso db tokens create <database-name>
```

Assign credentials to the environment variables inside `.env.local`.

```bash
VITE_TURSO_DB_URL="..."
VITE_TURSO_DB_AUTH_TOKEN="..."
```
</Step>

<Step title="Configure LibSQL Client">

```javascript IndexPage.vue
import {createClient} from '@libsql/client/web';

const turso = createClient({
  url: import.meta.env.VITE_TURSO_URL,
  authToken: import.meta.env.VITE_TURSO_AUTH_TOKEN,
});
}
```

<Warning>
To avoid running into a [gotcha moment], modify the Quasar app's configuration by adding the following settings to the build object.
</Warning>
```javascript quasar.config.js
...
build: {
  target: {
    browser: [
      'es2020', 'edge88', 'firefox78', 'chrome87', 'safari13.1'
    ],
    node: 'node16'
  },

  extendViteConf(config) {
    config.optimizeDeps = {
      esbuildOptions: {
        target: 'es2020',
      },
    };
  },
  ...
}
...
```

</Step>

<Step title="Fetch data from Turso.">

```javascript IndexPage.vue
import {ref} from 'vue';

const items = ref();

const { rows } = await turso.execute('select * from my-table');
items.value = rows;
```

</Step>

</Steps>
